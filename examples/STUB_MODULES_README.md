# Stub Modules Documentation

This document describes the stub modules created to make all examples compilable.

## Purpose

Many examples in this guide demonstrate integration with external systems, build-time code generation, or conceptual patterns. To make these examples compile and run without external dependencies, we've created minimal stub implementations.

## Stub Modules by Chapter

### Chapter 8: Build System (`ch08_build_system/`)

**`build_options.zig`** - Mock build-time configuration
- Simulates compile-time options typically generated by the build system
- Provides: `version`, `max_connections`, `enable_logging`
- **Note**: In a real project, this would be generated by `build.zig`

### Chapter 9: Packages & Dependencies (`ch09_packages_dependencies/`)

**`basic_math.zig`** - Simple math library stub
- Provides: `add(a, b)` function
- Demonstrates package imports

**`advanced_math.zig`** - Advanced math library stub  
- Provides: `pow(base, exp)` function
- Demonstrates conditional package imports

**`build_options.zig`** - Feature flags
- Provides: `advanced_enabled` flag
- Demonstrates build-time feature configuration

### Chapter 11: Interoperability (`ch11_interoperability/`)

**Inline C function mocks** - Simulated C library functions
- **01_basic_c_interoperability.zig**: Mocks `printf`, `strlen`, `malloc`, `free`, `memset`
- **02_sqlite3_library_integration.zig**: Full SQLite3 API mock
- **Note**: Real implementations would use `@cImport` with system C libraries

### Chapter 12: Testing & Benchmarking (`ch12_testing_benchmarking/`)

**`benchmark.zig`** - Benchmarking utilities
- Provides: `benchmarkWithArg()`, `compareBenchmarks()`, `BenchmarkResult`
- Simple timer-based benchmark implementation

**`snaptest.zig`** - Snapshot testing framework stub
- Provides: `Snap` type and `snap_fn()` helper
- Demonstrates snapshot testing patterns
- **Note**: Real version would persist/compare snapshots

## Using Real Implementations

To use real implementations instead of stubs:

### Chapter 8: Build System
```zig
// In build.zig:
const options = b.addOptions();
options.addOption([]const u8, "version", "1.0.0");
options.addOption(bool, "enable_logging", true);
exe.root_module.addOptions("build_options", options);
```

### Chapter 9: Packages
```zig
// In build.zig.zon:
.dependencies = .{
    .basic_math = .{
        .url = "https://github.com/example/basic-math/archive/v1.0.0.tar.gz",
        .hash = "...",
    },
}
```

### Chapter 11: C Interoperability
```zig
// Requires system C libraries installed
const c = @cImport({
    @cInclude("stdio.h");
    @cInclude("sqlite3.h");
});
// Link with: exe.linkSystemLibrary("sqlite3");
```

### Chapter 12: Benchmarking
Consider using real libraries like:
- [zbench](https://github.com/hendriknielaender/zbench) for benchmarking
- [zig-diff](https://github.com/ziglibs/zig-diff) for snapshot testing
